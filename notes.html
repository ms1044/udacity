Welcome to HTML!

HTML (Hypertext Markup Lanuage) is a reasonably straightforward language that helps computers take standard text and turn it into something more aesthetically appealing. It's assisted by CSS (Cascading Style Sheets), a script that can quickly format many facets of your code.

HTML works by using tags. Like any language, it seems to have a few quirks. Some tags, like <br>, operate alone to accomplish a formatting task (such as a line break). Other tags, like <p>, require a closing tag (such as </p>) and format everything in between the tags accordingly.

A Variety of Options

HTML also seems to have some built-in redundancy--something I hope to learn more about in the future. Some tasks can be accomplished using more than one tag:

  1. The <br> tag breaks your text to the next line, but so does the <p> tag. It turns out the <p> actually creates a box of sorts, which we learned about later.
  2. The <i> tag italicizes your text, but the <em> tag does the same thing.
  3. The <b> tag bolds your text, but so does the <strong> tag. The material hinted at a difference between formatting text and emphasizing content. I would like to better understand this.

The Big Lessson

My best takeaway for this lesson has been understanding the use of <div> tags. Imagine everything on your webpage broken into boxes, and then break each of those boxes into its own div. You can assign classes and id's to your divs, which helps the CSS identify which parts of your code to format. Once the important parts are divided into boxes, presumably CSS can parse and format those boxes in any number of ways.

Spans seem to be just as important as divs, but I haven't immediately seen many uses for them. It seems that a span would be helpful if you wanted to have several inline tags in one block of text, but again--I don't see that happening very often. I'm certain they'll become more useful to me as I learn more.

  • The <img> tag imports pictures from some reachable source--and, of course, should exist in its own div.
